# Conceptual Foundation

**Psiquis-X** was created to address the need for deterministic, secure, and auditable enterprise-grade agentic workflows. Below are the foundational principles regarding its architecture and infrastructure generation.

## 1. Deterministic Outcomes
The framework prioritizes accuracy over creativity. It employs an adversarial cross-validation design (the "Courtroom Architecture" with Judge and Skeptic models) to systematically interrogate sources, dispute hallucinations, and enforce factual reasoning.

## 2. Total Traceability & Data Lineage
Every piece of information ingested into the system carries a permanent, immutable digital footprint. As seen in modules like `pdf_intelligence.py`, source provenance (e.g., `[SOURCE_FILE: name] [PAGE: N]`) is strictly attached to all fragments, ensuring that the system's conclusions can always be traced back to raw data.

## 3. Stateful Long-Term Memory
Through persistent SQL (`psiquis_state.db`) and localized Vector memory (`chroma_db/psiquis_core`), Psiquis-X maintains context across multi-hop reasoning (DRIFT Search). It identifies macro-communities before retrieving micro-data, facilitating complex strategic synthesis over time.

## 4. Zero-Trust Security & Isolation
Core security includes local, transient secrets management (`agente_secrets_manager.py`) isolated from long-term memory. Additionally, any code generated by the AI (P-Series) is strictly confined to an execution sandbox (`genesis_sandbox.py`) with hard timeouts to prevent logic attacks or infinite loops.

## 5. Resilient Self-Healing
Error gracefully and recover intelligently. Scrapers (`data_interface.py`) fallback to cached data upon anti-bot blocks, while the code sandbox catches compiler errors and asynchronously prompts agents for rewrites, up to 3 automatic attempts, ensuring the system never dies mid-mission.

## 6. Real-World Execution (The "Hands")
Through universal MCP clients and an API-first approach (the `server_v3_minimal.py` entry point), Psiquis-X isn't just a chatbot; it runs background telemetry, writes Enterprise Excel reports programmatically, and actively operates external tools.
